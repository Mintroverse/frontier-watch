Obsolescence check

Is the in-house work still the right bet?

Editorial assessment — judgment, not source data

Short answer: the frontier is converging on the architecture already in place here, not replacing it.

The loudest theme this edition — from Gartner's framing to Microsoft's pitch to the data-readiness drumbeat in survey after survey — is that AI is only as good as the governed definitions and deterministic logic underneath it. Deterministic planning engines (order fulfillment, MRP, frozen-forecast compilation) plus a governed Fabric/Power BI semantic layer is exactly the substrate vendors now sell as the prerequisite for agents. That work is appreciating, not depreciating.

Validated by this edition

  • Deterministic engine + LLM assist as an architecture: Salesforce now ships a scripting layer (Agent Script) because reasoning models can't be trusted to enforce sequence — the same hybrid split used in in-house tooling.
  • Semantic-model-first: Fabric IQ builds its agent context layer directly on Power BI semantic models — existing DAX/model investment is the on-ramp.
  • Metric discipline: practitioner benchmarks show foundation forecasters ≈ well-engineered classical models on stable series; error-distribution measurement (WAPE/bias style) is exactly how the field says to evaluate them.

Watch — could overlap or absorb

  • Planning-in-Fabric (GA'ing now) writes forward-looking plans back into OneLake — potential overlap with custom capacity/WIP planning tools within 2–3 quarters.
  • Acumatica AI Studio + assistant could absorb some hand-built Power Automate / Generic Inquiry workflows once the Onematica instance reaches 2026 R1 — worth knowing the upgrade timeline before building more.
  • Manual orchestration of AI tooling is the pattern most at risk: the ecosystem (MCP Tasks, agent skills, service principals) is standardizing programmatic loops fast.
